Intruder

Burp Intruder is a powerful tool for automating customized attacks against web applications. It enables you to configure attacks that send the same HTTP request repeatedly, inserting different payloads at predefined positions.

Attack Types

üéØ Sniper Attack

Single payload set, one position at a time. Uses payloads sequentially at each marked position individually. Best for: Testing each parameter for common vulnerabilities (SQLi, XSS).

Example: Testing 3 positions with 5 payloads = 15 requests (5 √ó 3)

üî® Battering Ram

Single payload set, all positions simultaneously. Inserts the same payload into all positions at once. Best for: When the same input is required in multiple places (CSRF tokens, repeated parameters).

Example: Testing 3 positions with 5 payloads = 5 requests

üç¥ Pitchfork

Multiple payload sets, parallel iteration. Each position gets its own payload set, iterated in parallel. Best for: Correlated inputs like username/password pairs.

Example: 100 usernames + 100 passwords = 100 requests

üí£ Cluster Bomb

Multiple payload sets, all combinations. Tests every combination of payloads across all positions. Best for: Brute-force attacks with unknown correlations.

Example: 100 usernames √ó 100 passwords = 10,000 requests

Sequencer

Sequencer performs statistical analysis on session tokens and other important data items to assess their randomness quality. Weak randomness can lead to session prediction attacks.

Analysis Methods

üì°

Live Capture

Automatically capture tokens by repeatedly sending a request and extracting the token from responses.

üìÇ

Manual Load

Load pre-collected token samples from a file for offline analysis.

Randomness Tests Performed

  • Character-level analysis: Distribution of characters at each position
  • Bit-level analysis: Statistical tests on individual bits
  • FIPS 140-2 tests: Monobit, Poker, Runs, Long runs
  • Spectral analysis: Detecting patterns in the frequency domain
  • Correlation tests: Relationship between consecutive tokens

BChecks is a lightweight DSL (Domain Specific Language) for creating custom scan checks without writing Java code. Perfect for quick, targeted vulnerability detection.

metadata: language: v2-beta name: "Debug Parameter Discovery" description: "Checks for debug parameters that may expose sensitive info" author: "Security Team" tags: "passive", "information-disclosure" given response then if {latest.response.body} matches "(?i)(debug|trace|stack.?trace|error.?message)" then report issue: severity: low confidence: tentative detail: `Potential debug information in response body` remediation: `Remove debug output in production environments` end if
